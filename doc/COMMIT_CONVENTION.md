# Commit Convention

## 기본 형식

```
<type> subject

<body>

<footer>
```

---

## Type 종류

| Type       | 설명                                      |
|------------|-------------------------------------------|
| `feat`     | 새로운 기능 추가                          |
| `fix`      | 버그 수정                                 |
| `docs`     | 문서 작성 및 수정                         |
| `style`    | 코드 포맷 변경 (기능 변경 없음)           |
| `refactor` | 코드 리팩토링 (기능/버그 변경 없음)       |
| `test`     | 테스트 코드 추가 및 수정                  |
| `chore`    | 빌드, 설정 파일 등 기타 변경              |
| `revert`   | 이전 커밋 되돌리기                        |

---

## 규칙

- **헤더 형식**: `<type>` 형태로 꺾쇠괄호(`< >`)를 사용하여 Type을 표기
- **언어**: Commit Message(subject, body)는 반드시 **한글**로 작성
- **subject**: 50자 이내, 마침표 없이, 명령형으로 작성
- **body**: 변경 이유와 내용을 72자 줄바꿈으로 작성 (선택)
- **footer**: 관련 이슈 번호 기재 (선택, 예: `Closes #123`)

---

## 예시

```
<feat> 로그인 기능 추가

사용자 이메일/비밀번호 기반 로그인 API 연동.
JWT 토큰을 로컬스토리지에 저장.

Closes #42
```

```
<fix> 긴급 Fix
```

```
<docs> 커밋 컨벤션 초안 작성
```
