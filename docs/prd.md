# PRD: K-Stock-AutoTrader (한국 주식 자동 투자 프로그램)

## 1. 프로젝트 개요
본 프로젝트는 Python을 사용하여 대한민국 주식 시장에서 자동으로 주식을 매매하는 프로그램을 개발하는 것을 목표로 한다. 사용자의 개입 없이 설정된 전략에 따라 시장 데이터를 분석하고 주문을 수행한다.

## 2. 목표 및 핵심 가치
- **자동화**: 9:00~15:30 장 시간 동안 지속적인 모니터링 및 매매 수행.
- **안정성**: TDD를 통한 검증된 코드 기반으로 오작동 방지.
- **확장성**: 다양한 증권사 API 및 투자 전략을 쉽게 추가할 수 있는 구조.

## 3. 주요 기능 요구사항

### 3.1. 증권사 연동 (Open API)
- 증권사(예: 한국투자증권 KIS API) 서버와의 인증(OAuth2.0 등) 및 연결 유지.
- 접근 토큰(Access Token) 발급 및 자동 갱신 기능.

### 3.2. 시장 데이터 수집
- 특정 종목의 실시간 현재가 조회.
- 일봉/분봉 등 과거 차트 데이터 수집.
- 계좌 잔고 및 보유 종목 현황 조회.

### 3.3. 매매 로직 수행
- 설정된 투자 전략(예: 이동평균선 골든크로스, 변동성 돌파 등)에 따른 매수/매도 시그널 생성.
- 시그널 발생 시 즉각적인 주문(시장가/지정가) 실행.

### 3.4. 리스크 관리 및 로그
- 손절(Stop-loss) 및 익절(Take-profit) 기능.
- 모든 매매 내역 및 오류 로그 기록.

## 4. 기술 스택
- **언어**: Python 3.10+
- **라이브러리**: `requests`, `pandas`, `pytest` (TDD용)
- **API**: 한국투자증권 REST API (우선 적용)

## 5. 제약 사항
- 한국 주식 시장 정규 시간(09:00 ~ 15:30) 내 작동을 기본으로 함.
- 증권사 API의 호출 제한(Rate Limit)을 준수해야 함.

## 6. 단계별 개발 계획
1. **Phase 1**: 증권사 API 인증 및 계좌 정보 조회 기능 구현 (TDD 적용).
2. **Phase 2**: 실시간 시세 조회 및 기본 매수/매도 기능 구현.
3. **Phase 3**: 단순 이동평균선 전략 구현 및 자동 매매 루프 구축.
4. **Phase 4**: 예외 처리 강화 및 로그 모니터링 시스템 구축.
